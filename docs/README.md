# Docs

Lightweight progress-tracking docs for the pirate memory game. Keep every file short.

## Files

- `progress.md` — Current status: what works, what is partial, what is missing, known limits, and the current prototype goal. Check this before changing code.
- `decisions.md` — Locked-in design and technical decisions. Update when a decision changes.
- `next-steps.md` — Prioritized future work. Check this before changing code; update when priorities change.
- `changelog.md` — Short, dated list of meaningful changes. Add an entry after every meaningful code change.

## Workflow

1. Before changing code: read `progress.md` and `next-steps.md`.
2. After a meaningful change: update `progress.md` and `changelog.md`.
3. If a decision changes: update `decisions.md`.
4. If future work changes: update `next-steps.md`.

Keep it brief. Do not write long documentation.
